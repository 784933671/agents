# UI 修复手册

面向"已复现、待修复"的 UI 缺陷的实操指南。核心是：先定位根因，再用最小且行为保持的变更修复，最后验证回归面。

## 何时使用本技能

- 一个 UI 问题已复现，需要安全修复
- 修复时要避免引入新的视觉/行为回归
- 需要判断是"局部样式问题"还是"组件设计问题"

## 修复流程

```
1. 复现并锁定 → 2. 定位根因 → 3. 最小变更 → 4. 回归验证
```

### 1. 复现并锁定

- 明确触发条件（视口、状态、数据、操作序列）
- 截取或记录现场（DOM 快照、样式计算值）
- 确认这是"bug"还是"预期外但合理"的行为

### 2. 定位根因

按可能性排查：

| 层级 | 排查点 |
|------|--------|
| 布局 | Flex/Grid 配置、定位上下文、`min-width`/`overflow` |
| 样式 | 选择器特异性、token 误用、断点覆盖 |
| 状态 | 条件渲染、状态过期、竞态更新 |
| 数据 | 后端返回结构与渲染假设不符 |

工具辅助：
- DevTools Elements 面板查实际 DOM 和计算样式
- 临时禁用样式逐层定位
- 二分注释法缩小嫌疑范围

### 3. 最小变更原则

- **能改样式不改结构**：先用 CSS 解决
- **能局部不扩散**：不重构无关代码
- **能显式不隐式**：依赖和副作用写清楚
- **保持行为**：不破坏已有的正确路径

```css
/* ❌ 用 !important 强压，掩盖根因 */
.bad { display: block !important; }

/* ✅ 修正特异性，根因解决 */
.parent .child { display: block; }
```

### 4. 回归验证

- 复跑原缺陷，确认修复
- 检查共享 token / 组件的上游使用方
- 覆盖邻近状态（如修复了 hover，也要看 focus/active）
- 必要时补充一个回归测试

## 常见修复模式

### 布局错位

| 现象 | 根因 | 修复 |
|------|------|------|
| 内容溢出容器 | 固定宽度 + flex 收缩 | `min-width: 0` 或 `overflow` |
| Flex 项不等高 | 默认 `align-items: stretch` 被覆盖 | 恢复 stretch 或显式设置 |
| 绝对定位飘 | 缺 `position: relative` 父级 | 补定位上下文 |

### 状态相关 bug

| 现象 | 根因 | 修复 |
|------|------|------|
| 切换后状态错乱 | 未清理上一次的副作用 | useEffect 清理 |
| 闪烁/跳动 | 首次渲染与数据到达后不一致 | 骨架屏占位 |

## 反模式

- ❌ 未经定位就堆 CSS 补丁
- ❌ 改动超出缺陷范围（顺手重构）
- ❌ 修复后不验证邻近状态
- ❌ 用 `!important` 或内联样式掩盖根因
